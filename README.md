## 🎯 Задача
**Бинарная классификация:** на основе исторических данных о поведении клиентов банка (возраст, баланс, кредитный рейтинг, география и др.) необходимо предсказать, уйдет ли клиент в ближайшее время.

**Цель бизнеса:** Выявление клиентов с высоким риском оттока для своевременного предложения программ удержания. Удержание текущего клиента обходится дешевле, чем привлечение нового.

**Метрики:**
*   **ROC-AUC** — основная метрика. Выбрана из-за сильного дисбаланса классов (80% лояльных / 20% ушедших).
*   **Recall** — дополнительная бизнес-метрика. Наша цель — минимизировать False Negative (не упустить реально уходящих клиентов).

## 📊 Данные
Использован датасет **Churn Modelling** (Kaggle). 
*   **Объем:** 10 000 записей, 14 признаков.
*   **Целевая переменная:** `Exited` (1 - клиент ушел, 0 - клиент остался).
*   **Ограничения:** Наличие неинформативных признаков, которые были удалены на этапе предобработки. Присутствует дисбаланс целевой переменной.

## 🧠 Подход и Моделирование
**Validation Draft**: Данные разбиты на train и test в пропорции 80/20 с использованием стратификации (stratify=y) для сохранения баланса классов.

**Baseline:** В качестве базовой модели использована Логистическая регрессия. Для борьбы с дисбалансом применен параметр class_weight='balanced'. Baseline показал ROC-AUC = 0.77.

**Финальная модель**: Мы сравнили алгоритмы Random Forest и CatBoost. Победителем стал CatBoost с параметром auto_class_weights='Balanced'.

## 📈 Результаты и Валидация

Метрики качества CatBoost на тесте: ROC-AUC = 0.86, Recall = 0.70.

**Анализ ошибок**: Модель успешно выявляет 70% уходящих клиентов. Присутствует доля ложных срабатываний, однако с точки зрения бизнес-логики превентивное поощрение лояльного клиента несет меньше рисков, чем потеря уходящего.

**Анализ показал, что ключевыми факторами оттока являются:**
1. Возраст 
2. Количество банковских продуктов
3. Баланс на счету
